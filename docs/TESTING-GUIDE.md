# SimpleShop — Інструкція тестування

## Швидкий старт

```bash
# 1. Збірка та запуск
DOCKER_BUILDKIT=0 docker build -t simpleshop:local .
docker compose -f docker-compose.coolify-final.yml up -d

# 2. Встановлення + міграції + seed
docker exec simpleshop-app composer install
docker exec simpleshop-app php artisan migrate:fresh --seed --force
docker exec simpleshop-app php artisan storage:link

# 3. Відкрити
# Сайт: http://localhost:8088
# Адмін: http://localhost:8088/admin
```

**Акаунти:**
- Адмін: `admin@mail.com` / `123456`
- Клієнт: `user@test.com` / `password`

## 2. Картка товару та варіанти

### Тест 2.1: Сторінка товару
1. Відкрити будь-який товар
2. Перевірити: фото, ціна, SKU, бренд, опис
3. Вкладки: Опис, Характеристики, Відгуки, Доставка
4. Пов'язані товари внизу

### Тест 2.2: Варіанти товару
1. В адмінці: Товари → Edit товар → вкладка "Варіанти"
2. Додати опцію "Колір" (type: color), значення: Червоний (#FF0000), Синій (#0000FF)
3. Додати опцію "Розмір" (type: button), значення: S, M, L
4. Зберегти товар
5. В RelationManager "Варіанти товару" → натиснути "Генерувати варіанти"
6. Перевірити: 6 варіантів створено (3 кольори × 2 розміри або як налаштовано)
7. На фронтенді: відкрити товар → кольорові свотчі + кнопки розмірів
8. Клікнути різні опції → ціна, SKU, наявність оновлюються

### Тест 2.3: Варіанти в кошику
1. Вибрати варіант (колір + розмір)
2. Додати в кошик
3. Вибрати інший варіант того ж товару
4. Додати в кошик
5. Відкрити кошик → два окремих товари з різними назвами/цінами

## 6. Адмін-панель

### Тест 6.1: Вхід
1. Перейти `http://localhost:8088/admin`
2. Увійти як admin@mail.com / 123456

### Тест 6.2: Конструктор головної `/admin/homepage-builder`
1. Бачимо 9 модулів у списку
2. Натиснути ↑/↓ для зміни порядку
3. Toggle модуль off → перевірити головну (модуль зник)
4. "Редагувати" → змінити налаштування → зберегти
5. "Додати модуль" → вибрати тип → модуль з'являється

### Тест 6.3: Пакетний редактор `/admin/batch-editor`
1. Фільтр по категорії → товари відфільтровані
2. Змінити ціну inline → жовта підсвітка
3. "Зберегти зміни" → оновлено
4. Вибрати кілька товарів → "Ціна" → збільшити на 10% → Apply
5. "Акція" → встановити -20% → перевірити old_price/price
6. "Пошук/Заміна" → знайти текст → замінити
7. "Гуртові ціни" → вибрати групу Wholesale → встановити -5%
8. "Дублювати" → товари скопійовані з "(Копія)"
9. "Експорт" → CSV завантажується

### Тест 6.4: Інтеграції `/admin/integrations-page`
1. Бачимо 12 інтеграцій по групах
2. Toggle on/off → зберігається
3. "Налаштувати" → модалка з полями конфігурації
4. Telegram: ввести bot_token + chat_id → "Тест" → повідомлення в Telegram

### Тест 6.5: Групи клієнтів
1. Управління магазином → Групи клієнтів
2. Бачимо 4 групи (Retail, Wholesale, VIP, Distributor)
3. Відкрити Wholesale → вкладка Users → прив'язати користувача
4. Перевірити ціну на фронтенді для цього користувача

### Тест 6.6: Лояльність
1. Рівні лояльності → 4 рівні з множниками
2. Транзакції балів → "Коригувати бали" → вибрати user → нарахувати 500 балів

### Тест 6.7: Товари

---

## 9. Безпека

1. `curl -I http://localhost:8088/` → перевірити headers:
   - `X-Content-Type-Options: nosniff`
   - `X-Frame-Options: SAMEORIGIN`
   - `X-XSS-Protection: 1; mode=block`
   - `Referrer-Policy: strict-origin-when-cross-origin`
2. `/account` без auth → redirect на `/login`
3. `/admin` без is_admin → 403
4. 5+ спроб login → rate limited
